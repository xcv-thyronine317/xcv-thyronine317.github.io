📦 stackr - Simple PHP environment management for Windows

Download for Windows

stackr helps you run local web projects on your computer. It provides the software tools you need to build websites without complex setup procedures. You can manage databases, web servers, and email testing tools in one window.

💡 What is stackr?

This program acts as a central hub for web development. When you write code for websites, your computer needs special software to display the pages and store data. stackr puts this software in one place. You get Apache or Nginx for your server, PHP to run your scripts, and database tools like MySQL, MariaDB, or PostgreSQL. It also includes tools like Redis and Mailpit for advanced tasks.

🚀 Getting Started

Follow these steps to set up the software on your machine.

1. Download the Installer

Visit the official releases page on GitHub. Look for the latest version file that ends in .exe. Click the link to save the file to your computer.

2. Install the Software

Locate the installer file in your Downloads folder. Double-click the file to open it. If a security prompt appears from Windows, click "Run". Follow the on-screen instructions to finish the installation process.

3. Open the Application

Once installation finishes, an icon appears on your desktop. Double-click it to start the program. The dashboard opens and displays your current server status.

🛠️ Essential Features

  • Server Management: Start and stop your web servers with one click.
  • Database Tools: Choose between MySQL, MariaDB, or PostgreSQL for your projects.
  • Email Testing: Use the built-in Mailpit service to view emails sent by your scripts during development.
  • Cache Support: Enable Redis or Memcached if your projects require high-speed data storage.

⚙️ System Requirements

Your computer must meet these basic needs to run this program efficiently:

  • Windows 10 or Windows 11.
  • At least 4 gigabytes of memory.
  • Permissions to install software on your machine.

💬 How to get help

If you encounter issues, check the project page for common questions. You can also open an issue on the GitHub repository if you find a bug. Please describe the steps you took and include what you saw on the screen. This helps others fix the problem for you.
